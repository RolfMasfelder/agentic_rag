# Hybrid Agentic RAG System (On-Prem, Python/Django/PostgreSQL)

## Projektziel

Entwicklung eines lokalen KI-gestützten Analyse- und Retrieval-Systems für strukturierte und unstrukturierte Dokumente.

Das System soll:

* Dokumente und Sourcecode analysieren
* semantische Suche ermöglichen
* Beziehungen zwischen Dokumenten erkennen
* iteratives Retrieval unterstützen
* lokale LLMs verwenden
* vollständig On-Prem betrieben werden

Der Fokus liegt NICHT auf einem einfachen "Chat mit PDFs", sondern auf einem erweiterbaren Analyse- und Wissenssystem.

---

# Grundprinzip

Das System besteht aus mehreren Ebenen:

1. Dokumentverarbeitung
2. Wissensspeicherung
3. Retrieval
4. Agentische Orchestrierung
5. LLM-gestützte Analyse

Das LLM besitzt keinen direkten Datenbankzugriff.

Stattdessen benutzt ein Agent Werkzeuge (Tools/MCP), um:

* Daten zu suchen
* Beziehungen zu analysieren
* Nachrecherchen auszuführen
* Ergebnisse zusammenzuführen

# Chunking-Strategie

Chunking darf NICHT rein tokenbasiert erfolgen.

Stattdessen fachlich strukturieren:

* Paragraphen
* Vertragsklauseln
* XML-Blöcke
* Klassen/Funktionen
* API-Endpunkte
* Markdown-Abschnitte

Jeder Chunk erhält:

* Metadaten
* Dokumentreferenz
* Positionsdaten
* Embedding
* optionale Relationen

Kombination aus:

1. Vector Similarity

4. Relational Traversal

# Agentische Architektur

## Wichtiger Unterschied

Das LLM beantwortet nicht einfach Fragen.

Der Agent:

* plant Retrieval-Schritte
* führt mehrere Suchläufe aus
* bewertet Ergebnisse
* erzeugt neue Suchanfragen
* kombiniert Resultate

---

# Beispielhafter Ablauf

Userfrage:
"Welche Verträge enthalten ähnliche Haftungsregeln?"

Ablauf:

1. Agent analysiert Anfrage
2. Retrieval über pgvector
3. Ähnliche Klauseln finden
4. Metadaten filtern
5. Beziehungen traversieren
6. Weitere Retrieval-Schritte ausführen
7. Ergebnisse aggregieren
8. Antwort mit Quellen erzeugen

# Architekturprinzipien

Wichtig:

* Retrieval wichtiger als Modellgröße
* Datenmodell wichtiger als Chat-Oberfläche
* Nachvollziehbarkeit wichtiger als kreative Antworten
* Kleine spezialisierte Tools statt "magischer KI"

---

# Langfristiges Ziel

Kein einfacher Chatbot.

Sondern:

Ein lokales Wissens- und Analysesystem mit:

* semantischer Suche
* strukturellen Beziehungen
* iterativer Exploration
* agentischer Tool-Nutzung
* nachvollziehbaren Quellen
* kontrolliertem Datenzugriff

Das LLM dient dabei als Analyse- und Orchestrierungskomponente.
